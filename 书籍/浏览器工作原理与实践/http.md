### HTTP1.1

- 特性
  - 持久连接
  - 浏览器为 每个域名 最多同时维护 6 个 TCP 持久连接
  - 使用 CDN 的实现域名分片机制（为了并发下载）
- 问题
  - TCP 慢启动
  - 多条 TCP 连接竞争带宽
  - HTTP 的队头阻塞（同一管道）

### HTTP2

- 多路复用机制
  - 实现：二进制分帧层
  - 并发传送资源
- 特性
  - 一个域名只使用一个 TCP 长连接和消除队头阻塞问题
  - 设置请求的优先级
  - 服务器推送
  - 头部压缩
- 问题
  - TCP 的队头阻塞（按顺序接收数据包）

### HTTP3

- 网络延迟 RTT（Round Trip Time）
  - TCP + TLS 花费 3 ～ 4 个 RTT
- TCP 协议僵化
  - 中间设备的僵化
  - 操作系统更新慢
- QUIC 协议
  - 基于 UDP 协议（实现类似 TCP 的功能）
  - 集成 TLS
  - 实现了 TCP 级别的多路复用功能
  - 快速握手功能
- 问题（支持不够好）
  - 服务器和浏览器端
  - 系统内核
  - 中间设备
